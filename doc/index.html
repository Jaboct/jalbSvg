svg vs jsvg
(this is kinda like a white paper)
So there are a few goals to this.

1: Make a good vector editor, that I enjoy using for taking notes.
inkscape is a good svg editor, but its not good for taking notes.

2: be able to export what i make to svg.
for this project, and other projects like errandManager.

3: be able to procedurally generate svg elements.

4: have svg rendering tools, like line thickness and stuff.
probably stick this in draw2dApi.

5: I want CAD tools, first make in 2d eventually bring to 3d.


moving forward.
I will keep working on improving loading, saving, rendering of true svg files.
But right now I am interested in svg editing, and for that i feel like the svg structs are a little lacking, so lets start working on jsvg.




Nomenclature:
jalbSvg is the name of this project.
the mod jalbSvg is my version of SVG (standard vector graphics).

Then im going to make the mod jalbJvg (jalb vector graphics).

jvg can be exported to svg.
ideally svg can be imported to jvg.
